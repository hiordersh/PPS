# SKILL: ui-consistency — UI/UX 정합성 규정

> 하네스 ID: **H-02** | 적용 범위: PPS의 모든 페이지·컴포넌트 | 성격: **강제 규정**

## 목적

모든 페이지에서 스크롤바, 팝업, 버튼 위치, 레이아웃이 동일한 규칙을 따르게 하여
사용자가 어느 페이지에 있든 예측 가능한 경험을 제공한다.

---

## 규칙 1. 스크롤바 낭비 금지

- 페이지 스크롤은 **세로 1개만** 허용한다. 가로 스크롤은 어떤 페이지에서도 금지.
- 컨테이너 내부 이중 스크롤(스크롤 안의 스크롤) 금지. 목록이 길면 페이지네이션 또는 "더보기"로 처리.
- 콘텐츠 유무에 따라 스크롤바가 나타났다 사라지며 레이아웃이 밀리는 현상 방지:

```css
html {
  scrollbar-gutter: stable; /* 스크롤바 자리를 항상 예약 */
}
```

- 모달이 열릴 때 배경 스크롤은 잠그되(`overflow: hidden`), 스크롤바 폭만큼
  `padding-right`를 보정하여 화면이 좌우로 튀지 않게 한다.
- `overflow: auto`는 명확한 사유(예: 코드 블록, 테이블)가 있을 때만 사용하고 주석으로 사유를 남긴다.

## 규칙 2. 브라우저 기본 팝업 전면 금지 → 유저 팝업으로 대체

브라우저 네이티브 UI는 CSS로 제어할 수 없어 디자인 정합성을 깨뜨린다. 아래는 **사용 금지**:

| 금지 | 대체 컴포넌트 |
|---|---|
| `alert()` | `<Toast />` (알림) 또는 `<Modal />` (확인 필요 시) |
| `confirm()` | `<ConfirmDialog />` (확인/취소 버튼) |
| `prompt()` | `<InputModal />` |
| 기본 `<select>` 드롭다운 | 커스텀 `<Select />` (디자인 통일 필요 시) |
| 기본 툴팁(`title` 속성) | 커스텀 `<Tooltip />` |

- 공통 컴포넌트는 `src/components/common/` 에 **1회만 구현**하고 전 페이지에서 재사용한다.
- 페이지별로 모달/토스트를 새로 만들지 않는다. 필요한 변형은 props로 해결한다.
- 예외: 브라우저 정책상 대체 불가한 것(파일 선택 다이얼로그, `beforeunload` 경고)만 허용.

## 규칙 3. 버튼 위치·레이아웃 고정 배치

모든 페이지는 아래 지정 위치를 따른다. 위치를 바꾸려면 이 스킬을 먼저 수정하고 컨펌받는다.

```
┌──────────────────────────────────────────────┐
│ Header: [로고(좌)] [내비게이션(중)] [검색·로그인(우)] │  ← 고정(sticky)
├──────────────────────────────────────────────┤
│ 페이지 타이틀(좌)              [주요 액션 버튼(우)] │
│                                              │
│              콘텐츠 (max-width 1200px 중앙)     │
│                                              │
├──────────────────────────────────────────────┤
│ Footer: 저작권(좌)              GitHub 링크(우)  │
└──────────────────────────────────────────────┘
```

- **주요(긍정) 액션**: 항상 **우측** — 저장, 등록, 확인
- **보조(부정) 액션**: 주요 액션의 **왼쪽** — 취소, 닫기
- 모달 버튼도 동일: `[취소] [확인]` 순서, 우측 하단 정렬
- 뒤로가기/목록으로 버튼: 페이지 타이틀 **좌측**
- 삭제 등 파괴적 액션: 빨간색 계열 + 반드시 `<ConfirmDialog />` 를 거친다
- 카드 그리드: 데스크톱 3열 / 태블릿 2열 / 모바일 1열 (브레이크포인트 1024px / 768px)

## 규칙 4. 인터랙션 통일성

- 색상, 간격, 라운드, 그림자, 전환 시간은 **CSS 변수(디자인 토큰)** 로만 사용한다:

```css
:root {
  --color-primary: #...;
  --radius-md: 8px;
  --space-unit: 8px;        /* 모든 간격은 8px 배수 */
  --transition-fast: 150ms ease;
  --transition-base: 250ms ease;
}
```

- 같은 의미의 인터랙션은 전 페이지에서 같은 방식으로 동작한다
  (예: hover 효과, 로딩 스피너, 빈 목록 표시, 에러 표시).
- 로딩/빈 상태/에러 상태는 공통 컴포넌트(`<Loading />`, `<Empty />`, `<ErrorBox />`)로 통일.
- 레이아웃 울렁거림(layout shift) 방지 세부 기법은 **H-03 `make-interfaces-feel-better.md`** 를 따른다.
