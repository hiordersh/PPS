# HARNESS.md — PPS 작업 하네스

> 모든 작업 규정과 스킬을 **모듈 단위로 등록·관리**하는 중앙 인덱스.
> 궁극적 목표: **재귀적 자가학습 / 루프엔지니어링** — 작업할수록 규정과 스킬이 축적되어
> 다음 작업의 품질과 속도가 자동으로 향상되는 구조.

---

## 1. 루프엔지니어링 사이클

모든 작업은 아래 루프를 따른다:

```
┌─────────────────────────────────────────────────────┐
│  ① 계획   : 작업 범위 정의 (직전 작업까지만 참조)     │
│  ② 탐색   : github.com/trending 에서 활용 가능한      │
│             오픈소스/스킬 우선 검토                    │
│  ③ 구현   : 하네스에 등록된 규정/스킬을 준수하며 작업  │
│  ④ 검증   : 규정 위반 여부 자가 점검 (체크리스트)     │
│  ⑤ 규정화 : 작업 중 발견한 개선점을 스킬(.md)로 작성   │
│             → skills/ 에 추가, 본 하네스에 등록        │
│  ⑥ 커밋   : 산출물 + 신규/수정 스킬을 함께 커밋       │
└──────────────────── 다음 작업으로 ──────────────────┘
```

**원칙**: 같은 실수/판단을 두 번 하지 않는다. 두 번 이상 반복될 판단은 반드시 규정화한다.

## 3. 스킬 작성 규약

- **위치**: `skills/스킬명.md` (kebab-case)
- **구조**: `목적 → 규칙(강제) → 코드 패턴 → 체크리스트 → 출처(있는 경우)`
- **성격**: 규칙은 "권장"이 아닌 **강제**로 기술한다. 예외는 예외 조건과 함께 명시.
- **모듈화**: 한 스킬은 한 주제만 다룬다. 주제가 섞이면 분리한다.
- **출처 표기**: trending 등 외부에서 가져온 경우 원본 저장소/문서 링크를 남기고,
  내용은 PPS 맥락에 맞게 재작성한다 (원문 통복사 금지).

## 4. 작업 참조 규칙 (사용자 지정)

- 모든 작업은 **직전 작업까지만 참조**한다.
- 전체 재검토 / 처음부터 재작업은 사용자가 명시적으로 지시하고 **컨펌한 경우에만** 수행한다.
- 새 제안은 사용자에게 추천 → 컨펌 후 진행한다.
